OpenHoldem:EndUserDocumentation:Gotchas:no valid tables found

From Maxin Wiki

Jump to: navigation, search

OpenHoldem will report this for one of the following reasons

  • Your system settings are incorrect. Check this first: System_settings
  • No poker table is visable on your desktop.
  • The corresponding TableMap (*.tm file) is not present in the "scraper" sub-folder of your OpenHoldem install folder (First create a folder in one of your drives (eg: C Drive) and call it OH or Open Holdem. Keep all the programs in this file and create a new folder within this one called "scraper". This folder will be where you save all your table maps. You can create a shortcut to your desktop if you like, it wont affect anything)
  • The poker table does not meet all the requirements set out in the .tm file.
- clientsize (details to check this are below)
- titletextX (string from record must be present in the Poker window title)
- !titletextX (string from record must not be present in the Poker window title)
- tablepointX (every tablepoint region must resolve true or OH will not connect)

The records above can be found in the .tm file by opening with OpenScrape or right-clicking the file and "open with..." Notepad.

You can prevent OH from checking any or all of these conditions by commenting out the record , this is useful to find the culprit and therefore where the problem lies.

eg

Not working

 s$titletext1      Tournament: # Table
 s$titletext0      !
 s$titletext       Hold'em

Try

 //s$titletext1      Tournament: # Table
 //s$titletext0      !
 s$titletext       Hold'em

Checking clientsize

Start Openscrape

Press Green circle

Double click correct window
