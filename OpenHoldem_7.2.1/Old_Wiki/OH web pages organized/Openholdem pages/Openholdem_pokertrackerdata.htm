How to implent Poker Tracker data into Open Hold'em!


To use PokerTracker data into Open Hold'em there have to be done some work first. Make sure you use Pokertracker2.0 and not the new 3.0 version (as for May 2012 only Players auto-rate Icons can be retreived from PT3)!


Step 1: Download PostgreSQL, be sure to have the latest version at http://www.postgresql.org/download/


Step 2: Install the package and make sure you also install the ODBC package which is included in PostgreSQL, don't forget to write down the password you set for user postgres


Step 3: Restart your computer just to be sure


Step 4: Open Pokertracker 2.0 and go to database options, select convert Pokertracker Access Database to PostgreSQL database


Step 5: You will be asked to give the database a name, write down this name


Step 6: Start OpenHoldem and go to Edit/Preferences


Step 7: The next information will be asked to enter:


- HOSTNAME: When you run it without VMware on your pc you can leave 127.xxx.x.xxx, Otherwise enter the IP adress from your Virtual Machine, like 192.168.1.x

           To find the IP adress of your virtual machine you open the command prompt at the VM and you type "ipconfig" to find your new IP adress

- PORT: 5432 (also already filled in)

- USERNAME: postgres (standard use postgres, probably already filled in)

- PASSWORD: The password you setup in step 2

- DBNAME: The name of the PokerTracker database you gave it in step 5


Step 8: Table Map vars sitename and network are correcly set. For example if your network is iPoker set both to "iPoker". Beware of cases.


Step 9: Check that Players names are scraped correctly (name are needed for PT queries)


Step 10: Check that AutoPlayer is enabled, It's your turn, action buttons are active and you got hole cards



Now that you have done this you can test this by opening a real money table and try to connect to PokerTracker by pressing Test. This should confirm you have access to the database. When you run Poker Tracker on your virtual machine you have to install PostgreSQL on your virtual machine too and have to follow the Pokertracker instruction's for setting up a remote database, more information about that can be found on the PokerTracker forum's.
