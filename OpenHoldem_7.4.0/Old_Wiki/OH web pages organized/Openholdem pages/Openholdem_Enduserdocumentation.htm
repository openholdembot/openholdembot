[edit] Overview over OpenHoldem

OpenHoldem (OH) is an opensource botting platform. OH is a bot provided only with "hands" and "eyes", this means that you will have to write his brain to make it to do what you want. There are three ways to do this:

  • Built in script capability
  • Using Perl
  • Using DLL (C++)

The decision is always directly based on OH formulas evaluation (script):

  • First evaluated is f$allin. If it's true it goes all-in, if not...
  • Then f$swag. If it's true, OH swags, if not...
  • OH evaluates f$raise. If it's true, OH raises, if not...
  • Then f$call. If it's true, OH calls, if not...
  • Finally, if all are false, It tries to check. If checking is possible then OH checks, if not...
  • OH folds

Using the dll you can influence these formulas, creating symbols that OH will use to evaluate them. There is a template to start developing the DLL on this Wiki. To see what's happening on the table OH uses screen scraping so all tables' windows have to be visible and not occluded by any other window. For OH to be able to 'read' the table, it needs a table map file that tells it where to find info and how to extract them, and can be created with OpenScrape .


Source(s):OpenHoldem


[edit] Default Table map

When the green button (connect) is pressed, OpenHoldem looks for table maps in a "scraper" sub-folder. It will select the applicable TM's and if there are many of them, it'll open a dialog to select the right one[1].
