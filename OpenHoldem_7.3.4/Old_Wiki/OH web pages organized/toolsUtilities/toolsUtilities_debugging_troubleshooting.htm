Debugging / Troubleshooting

From Maxin Wiki

Jump to: navigation, search
  • Windows_Debugger_Installation_Script

- Silently install WinDbg and set it as default JIT debugger. Having WinDbg installed will catch OH crashes. You can then use another script available in this section to automate sending mini dump to developers, hopefully helping them find and fix the problem.


  • Mini Dump Handling Script

- Waits for windbg window to popup in case of OH crash, initiates creation of mini dump file, copies file to central location, deletes dump folder and closes windbg window. Run it as resident or incorporate into your hopper.


  • Crash Test Utility

- This zip contains crash_test.exe and its source - it will intentionally crash when you start it, useful for testing the setup of debugger scripts. Generously provided by OpenHoldem.


  • Manual process for collecting minidumps


  • MiniDump Tool (from here)

- Can possibly be used to collect minidumps without all the above hassle. This is largely untested...please check the author's web site for instructions.
