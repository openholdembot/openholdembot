OpenScrape:About

From Maxin Wiki

OpenScrape is intended to be an OpenSource replacement for the WinScrape tool, as originally coded by Ray Bornert: http://forum.winholdem.net/wbb/viewforum.php?f=29

It is one of the few remaining tools that needed to be replaced to fully gain freedom from WinHoldem.


Source(s):OpenScrape


[edit] Purpose

The purpose of OpenScrape is to create "table maps", which in the most general form is a collection of rectangle definitions that instruct the scraper engine in OpenHoldem how to interpret the various user interface components of a casino table. You can think of the scraper engine in OpenHoldem as a general purpose Texas Holdem screen scraper (http://en.wikipedia.org/wiki/Screen_scraping), the behavior of which is determined by the parameters contained in the file created by OpenScrape.

OpenScrape will ultimately create a file with the ".tm" extension. This file is then used by OpenHoldem when you "click the green button" or choose "File/Load Profile" from the main menu.

The main benefit of a parameter driven scraping engine is that it frees OpenHoldem from being casino specific. Many poker bot programs/frameworks are quite limited in that they officially "support" only a handful of poker rooms. With end-user defined parameter files, it is theoretically possible that any and every poker room on the planet can be interpreted and engaged with OpenHoldem.
