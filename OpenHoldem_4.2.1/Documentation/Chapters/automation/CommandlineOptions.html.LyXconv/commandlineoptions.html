Command Line Options

Version 2.0.0 of OpenHoldem introduced the ability to instruct OpenHoldem to use .ini files for storage of preferences, rather than using the registry. This is beneficial for two reasons:

  • Stealth: .ini files are easier to hide than the registry.
  • Portability: .ini files can easily be deployed and moved from machine to machine.

Using the .ini-file

To specify the use of .ini files for preferences, rather than the registry, there are three options: “/ini” specifies the use of openholdem.ini, created in the OpenHoldem startup directory “/ini:[filepath]” specfies the use of [filename] as the ini file, must be a full path “/ini:~\[filename]” specifies the use of [filename] in the OpenHoldem startup directory as the ini file.

Creating the .ini-file

To create the .ini-file the first time start OpenHoldem with the “/ini”-parameter at the command-line, then open the preferences, click through all tabs and finally confirm with “OK”. OpenHoldem will then save all settings to the file instead of storing them in the registry.
