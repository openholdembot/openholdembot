[edit] Compilation Instructions

To compile OpenHoldem, you need a modern C++ compiler that supports expression templates and static polymorphism. OpenHoldem was compiled with Visual Studio 2005. Anything earlier likely does not provide the capabilities to compile templated C++ code properly. Anything of a later vintage than VS2005 (VS2008, for example) does not appear to be supported properly by the Boost Spirit library and will result in compiled code with severe performance problems. Non-Microsoft compilers have not been tested and are not supported.

- The Boost Spirit library needs to be properly installed. See http://boost.org/ for more information. Not needed as of OH v1.8.0

- If you wish to use Dan Moulding's Visual Leak Detector, then go get it from http://dmoulding.googlepages.com/vld and install it properly, otherwise comment out this line in debug.cpp:

 #include <vld.h>

- [The GMTL library needs to be properly installed. You can find that here: http://ggt.sourceforge.net/] Not needed as of OH v1.0.5

- [The PokerSource library needs to be properly installed. From here: https://gna.org/projects/pokersource/] Not needed as of OH v1.0.3

- The VC++ directories need to be set up to the installation paths for the above bulleted items. In VS2005, go to Tools/Options/Projects & Solutions/VC++ Directories. Under "Include Files" and "Library Files", change the paths to point to *your* installation paths.

All other open source code packages are included in the source code archive and are already configured properly.

Optional: If build in debug-mode, OpenHoldem will create a dump-file, if it encounters a fatal error.

[edit] SVN Repository Configuration

The Google Code SVN repository is configured as described here: http://code.google.com/p/openholdembot/source/checkout

[edit] Compiling OH with Perl support

The OpenHoldem project contains a fixed header for PerlEz.Dll and does not need PerlEz.lib for linking (as we link against Perl at runtime and not at loadtime). Therefore developers do not need to have Perl installed or take care about any special settings.
