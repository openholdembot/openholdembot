[edit] Poker Hand Generator

The Poker Hand Generator generates replay files for the Universal Hand Replayer

[edit] What is it for?

You can use it for regression testing your bot.
Example: You want the bot to raise 4xBB preflop first in, in any position with AA or KK.
Then you can create 20 hands, one hand for each position in a 10-handed ring game with AA and one hand for each position with KK.
You can then connect your bot to the URP table (in manual mode) and check if it behaves well whenever you have made changes to the preflop logic.
A nice extension to this would be if the OH software could be extended to read the replay files and perform these tests automatically.

[edit] How to use it

Download the executable here (Windows). You will also need.NET Framework 3.x to run the program.


Select Menu File/New. This will create a new Session. You will get a dialog where you can select

  • Folder
  • Session/file name
  • Number of chairs
  • Player names.

Player number 1 is always you (Hero), but in each hand you can select your table position.
The number of chairs is the maximum you can use in one session, but in each hand you can select a smaller number of players to participate.
Press OK, and you are back in the main window, where you press the Add New Hand button. This will bring up the New hand dialog, where you enter.
• Number of players (must be equal to or less than the number of chairs for the session)
• Your position (1= SB, 2= BB etc)
• Cards: your pocket cards and all community cards. There is a button for generating random cards.
• The sizes of the blinds
• Initial stack sizes for all players
• A comment describing the hand
When all data is entered (Comment is not needed), you can press the Betting actions button, which enables the Betting actions tab pages, one for each betting round.
Each betting round consists of one or more “table rounds” (I don’t know the correct term for this). Each table round will be one column of Action boxes in the Betting action tab page. The program will always set focus on action box for the player who is next to act when a new table round (or betting round) is created. You enter the action and the press the tab key to go to the next player. If no more players can act, a new betting round is created. The actions you can enter are:
• A number, representing your total bet on the betting round
• the letter “c” for check or call
• the letter “f” for fold
• the letter “p” for a pot-sized bet
• the letter “a” for an all-in bet
• p13, p12,p23, p34 for betting 1/3, ½, 2/3 or ¾ of the pot
When you don’t want to enter more actions, hit the “OK” button, and you are back to the main form again, where some data of your newly entered hand will be displayed in a table. You can choose to add more hands, or save the session file.
In the hands table, you can right click on any hand to delete or copy it. Copying means to copy the cards mainly. You can also add a new hand this way, but it will always become the last hand in the list.
The hands will always be sorted on the "Hand number" when saving the session.

[edit] Known problems

  • If you tab to the next action box without entering any action, the actions will come out in the wrong order on the session file.
  • The bet size check is rudimentary; I am not too sure on the rules here.
  • Illegal input check and recovery from errors caused by illegal input is rudimentary.
  • The only documentation is this Wiki, and som built-in help texts.
