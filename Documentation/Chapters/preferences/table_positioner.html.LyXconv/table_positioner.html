OpenHoldem Preferences - Table Positioner

Table Positioner preferences control if and how OpenHoldem arranges poker-tables at the screen
  • Tiled: this option will align all tables nicely without any overlap, so that OpenHoldem can see and scrape all tables clearly.
  • Cascaded: this option will stack the tables with lots of overlap, but usually in a way that a little piece of every table is visible. A useful option for some crazy human multi-tablers, but not yet for OpenHoldem, as scraping overlapping tables does at the moment only work on some operating-systems.
  • Never: this option will disable the table positioner and leave it to the casino software or to your hopper to align the tables in a reasonable way
  • Always keep position: this option will check once per heartbeat if the table is at its assigned position and move it back if needed.
The first table (or the lobby if you provide a “tablemap” for the lobby) will be moved to top-left of the screen, the second table will be moved to botto-right, and all further tables to the neighbourhood of the second one. Once the screen is full any additional tables will be moved to the lobby position, assuming that overlapping the lobby will do the least harm.
