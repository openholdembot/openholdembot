[edit] How to get started

Originally posted by Leon here (very first post), this was so good as to justify a sticky and a wiki entry. This is a great approach that everyone new to OpenHoldem should consider taking.

Hello all,

I've been lurking here now for a few weeks, absorbing knowledge and trying to build a Poker bot. I thought it about time to introduce myself and share what I have learnt with noobs out there who may just be starting out as I recently was.

Introduction - my name is Leon, I'm English but currently live in Barcelona and I'm a web designer/developer. http://www.outlet4louboutin.com/ Web stuff is OK, but has got a bit boring of late so I'm exploring other revenue streams, poker being one of them.

What I've learnt so far:

  • The language that the default bots are written in at first looks weird, especially if you're used to something like PHP or Perl. But actually it's very simple:
  1. f$rais, f$call and f$alli must evaluate to either a 1 or a 0 (1 means you raise/call/allin, 0 means you don't)
  2. f$srai and f$swag must evaluate to positive numbers (the amount you want to raise, and the amount you want to type into the Poker site's raise box. Watch out: these can be different) (Thanks for the correction OpenHoldem)
  3. f$P should evaluate to the number of opponents remaining in the hand.
  4. && means AND
  5. || means OR
  6. condition ? valueIfTrue : valueIfFalse is what's called a Ternary Operator. It takes a while to be able to read it easily. I still have to stop and think, "which part is true, which false?" It's especially hard to read when it's nested.

However, easy as it may be after a while, reading Perl syntax is still much easier. For this reason, and other advantages, I decided to write my bot in Perl. If you're getting lost in the syntax, you might want to do the same.

  • Installing Perl is easy, just follow the instructions here The only Gotcha I had was that after telling OpenHoldem to use Perl in the preferences you need to restart OpenHoldem. Ed: To be fixed in upcoming July 25, 2008 release'
  • You need somewhere to test your bot. You can do this in Manual Mode (don't forget to click the F and C buttons bottom right so OH recognises you as playing), but the best place I found (as recommended by others on the forum) is Poker Academy. You can rattle off a test of around 7000 hands in a night. Beware though, you may find some strange things happening..
  • ... because you shouldn't trust the table maps you find on the Wiki. People who upload table maps to the Wiki are very generous and give you a great start, but you really need to make sure they are working OK yourself. I found that moving from one PC to another could cause problems as the colour of the Pixels would be slightly different. So you need to verify that all the buttons are recognised (OH should at least have F,C and R highlighted at the bottom), that the balances are correct, that the blinds are correct. You may have to add in some fonts.
  • When your first revision of the bot is up and running (at say Poker Academy) you'll see it make mistakes (i.e something you think it shouldn't be doing) all the time. You need to know why, and in order to check all that, you need to make sure you are logging everything. Here's how I'm doing my logging:
  1. I am using Perl with Debug.pm, with this line removed: print (DEBUG scalar(gmtime())); and the name of the main function changed to doprint
  2. I have pl_call run a function called bl_write_game_state which contains the following code:
	my $logoutput;

	$logoutput = ('---START----') . "\n";
	$logoutput .= scalar(gmtime()). "\n";
	
	$logoutput .= ('OHS---->'). "\n";
	foreach my $key (sort keys %OHS) {
		if ( $key !~/currentbet[0-9]/ ) {
 		$logoutput .= '['.$key.'=>'.$OHS{$key}.']'. "\n"; 
		gws('log$'.$key.'___'.$OHS{$key});
		}
	}
			
	$logoutput .= ('vars---->'). "\n";
	foreach my $key (sort keys %vars) {
 		$logoutput .= '['.$key.'=>'.$vars{$key}.']'. "\n"; 
		$vars{$key} =~ s/[^\w]/_/g;
		gws('log$'.$key.'___'.$vars{$key});
		$vars{$key} = "";
	}
		
	$logoutput .= '[Pocket 1=>'.int(gws('$$pr0')).']'. "\n";
	$logoutput .= '[Pocket 2=>'.int(gws('$$pr1')).']'. "\n";
	$logoutput .= '[CC1=>'.int(gws('$$cr0')).']'. "\n";		
	$logoutput .= '[CC2=>'.int(gws('$$cr1')).']'. "\n";
	$logoutput .= '[CC3=>'.int(gws('$$cr2')).']'. "\n";
	$logoutput .= '[CC4=>'.int(gws('$$cr3')).']'. "\n";		
	$logoutput .= '[CC5=>'.int(gws('$$cr4')).']'. "\n";		
	
	$logoutput .= ('---------END--------'). "\n";
	
	doprint($logoutput);

This writes out all my cached OpenHoldem symbols to the debugging text file, and also to the OpenHoldem log file. Just make sure than in the OpenHoldem preferences you have "Enable Log Symbols" set and increase the number of max symbols. Also, any variable I use in my code I store in a hash called $vars. All these variables are also written to the logs.

Finally, in OpenHoldem preferences, enable autoplayer trace for functions you are interested in and in Replay Frames set it to record replay frames for every change in table state. You can play back the replay frames via OHReplay, which really helps to debug a situation when you can't understand why OH acted as it did. The only slight gotcha I found with replays is that sometimes the prwin isn't calculated - it thinks it's zero. I found moving the window around a bit will normally jog OH into calculating the prwin. Ed: Or, partially obscuring the OHReplay window with another window should trigger a recalc

  • Talking about prwin, don't put too much faith in it in all situations. It doesn't take into account opponent hand ranges (look into weighted prwin).
  • OpenHoldem doesn't choose tables for you. A program that chooses which table to play at is called a "Hopper". You're going to have to program one yourself, and this is difficult. You should start by downloading the free AutoIt, and reading the AutoIt help file. This guide is also useful reading. Note that an attempt was made at programming a general hopper on these forums, but it's been discontinued, I think due to the difficulty of pleasing everyone. The code is still available here though. So far I've coded a hopper for Poker Academy cash games and Sit N Gos, and it's great way to learn coding the hopper.

I think that's about it. Any questions shoot, and if some old hands found holes in my statements above please let me know.

Cheers,

Leon
