  1. Download and install the debugging tools here
  2. Configure windbg as a JIT (just-in-time) debugger

From Start/Run, assuming you installed in the default location: 

"C:\Program Files\Debugging Tools for Windows (x86)\windbg.exe" -I

At this point you have two choices

  • Choice 1: symbols/source
  1. Copy the symbols file and source code somewhere to the machine that is running OH
  2. Start windbg, and configure the symbol file and source paths appropriately (File/Symbol File Path, and File/Source File Path) This can be the same directory, BTW.
  3. Exit windbg, run OH. When OH crashes, windbg will come up and tell you exactly where it crashed.
  • Choice 2: minidump
  1. Run OH.
  2. When it crashes, windbg will come up
  3. Open a command prompt at the location where you installed the debugging tools. By default this is: "C:\Program Files\Debugging Tools for Windows (x86)"
  4. At the command prompt type
cscript adplus.vbs -quick -pn openholdem.exe

You may need to change openholdem.exe to whatever name you have OH running as.

  1. The dmp file will be located under the debugging tool directory in a folder that looks something like: "Hang_Mode__Date_06-01-2008__Time_10-00-00PM".
  2. Get the .dmp file to a developer.
